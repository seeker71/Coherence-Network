# Coherence Network — Plan

## Objective

Deliver a reliable, continuously improving OSS intelligence organism whose
runtime center is Form-native by prioritizing:

1. Form-native route/runtime correctness,
2. high-grammar handler coverage for web-used `/api` traffic,
3. spec coverage and proof traceability,
4. pipeline throughput and recovery,
5. operational clarity for maintainers.

## North Star

Use the body's own introspection to reveal repeated low-level recipes,
especially on hot paths, and lift them into simpler, more generic Form/BML
teachings: blueprints, grammars, recipes, cells, and proof bands that are
fractal enough to serve the next occurrence without adding a special case. The
plan is a direction, not a fixed endpoint: every route trace, JIT miss,
wellness signal, carrier-tissue read, and repeated edge category can pull the
core toward more elegant, flexible, adaptable, sovereign runtime tissue.

## In-Scope Work

### Product Delivery
- Form-native kernel front door and BML/domain handler promotion
- Project indexing and graph expansion
- Search and project/coherence endpoints
- Import stack analysis workflows
- Web/API integration for current shipped capabilities
- Public deployment maintenance (Hostinger API + Hostinger web)

### Pipeline Delivery
- Automated task routing and execution
- Monitoring and attention heuristics
- Retry/heal workflows
- Status reporting and runbook quality

### Quality Guardrails
- Spec-first implementation
- Test-first behavior
- CI health and deterministic checks
- Minimal, scoped diffs
- Introspection-led core lifting: repeated low-level shapes become named,
  reusable Form/BML abstractions with proof

## Current Milestone Track

- **Milestone A: Core Stability**
  - keep the Form-native route surface, compatibility API, and indexers stable
  - maintain production deployment health (Hostinger API + web)
  - reduce repeated pipeline failures
  - keep status and coverage docs accurate

- **Milestone B: Execution Efficiency**
  - move high-frequency handlers into BML/domain grammar routes
  - improve success rate and cycle time
  - improve monitor signal quality
  - reduce manual intervention per overnight run

- **Milestone C: Data/Intelligence Depth**
  - deepen Form-native persistence, JSON, and observation/JIT paths
  - extend graph data quality
  - improve coherence signal quality
  - tighten test/spec mapping for new capabilities

## Prioritization Rules

When tradeoffs appear, prioritize in this order:

1. Form-native correctness and testability,
2. hot-path simplification through reusable core abstractions,
3. route/runtime observability and maintainability,
4. pipeline reliability,
5. new feature breadth.

## Operating Principles

- Keep changes small and verifiable.
- Prefer Form/BML/domain grammar handlers over new compatibility endpoint logic.
- When a hot path or repeated low-level recipe appears, ask which generic
  teaching wants to exist before adding another local branch or helper.
- Prefer explicit specs over implied behavior.
- Do not modify tests to force pass conditions.
- Keep documentation aligned with shipped behavior.
