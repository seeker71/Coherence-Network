# Field Model Form

Field Model Form is the current grammar for work that happens over fields rather than strings. It keeps Blueprint, Recipe, and Cell, then adds carrier, topology, fiber values, observer cost, consent, residuals, and receipts.

## Shape

- **FieldBlueprint** names the field carrier, topology, boundaries, units, and scale.
- **FieldCell** carries the current sites, edges, traces, and active recipes.
- **FieldRecipe** observes, matches, prices, emits deltas, resolves conflicts, commits, and records receipts.

## Execution

snapshot -> match -> price -> emit delta -> resolve -> commit -> receipt + residual

Forward and reverse paths stay part of the same shape. A field change is not complete without the evidence that shows what changed, what remained unresolved, and how the path can be walked back.
