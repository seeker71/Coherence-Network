# Spec: Geolocation awareness ŌĆö nearby contributors, local ideas, regional news

## Summary

Coherence Network should make **place** a first-class lens: contributors can share coarse location safely, the API resolves human-readable places to coordinates via a **borrowed GeocodingService pattern** (OpenCage primary, OpenStreetMap Nominatim secondary, graceful degradation), and clients can **discover nearby collaborators**, **ideas tied to local contributors**, **regional news**, and eventually **tasks** scoped by geography. Today the API already exposes city-level contributor storage, haversine **nearby** search, and **keyword-based local news resonance**; this spec closes the gap to production-grade geocoding, richer profile surfacing, task proximity filters, web discovery surfaces, and **observable proof** that the feature works and improves over time.

## Purpose

- **Who benefits**: Contributors seeking local collaboration; operators proving regional engagement; reviewers validating deploys with concrete curl scenarios.
- **Failure avoided**: Silent ŌĆ£deadŌĆØ location features ŌĆö mitigated via explicit health, metrics, and versioned geocoder behavior.

## Current state (baseline)

The following already exist under `prefix="/api"`:

| Capability | Method | Path |
|------------|--------|------|
| Set location | `PATCH` | `/api/contributors/{contributor_id}/location` |
| Read location | `GET` | `/api/contributors/{contributor_id}/location` |
| Remove location | `DELETE` | `/api/contributors/{contributor_id}/location` |
| Nearby contributors + ideas | `GET` | `/api/nearby?lat=&lon=&radius_km=&limit=` |
| Local news (text resonance) | `GET` | `/api/news/resonance/local?location=&limit=` |

Implementation references: `api/app/routers/geolocation.py`, `api/app/services/geolocation_service.py`, `api/app/models/geolocation.py`.

**Gap**: Forward geocoding (address/city ŌåÆ lat/lon) is not yet centralized as a Living CodexŌĆōstyle **GeocodingService** with OpenCage + Nominatim + fallback. Contributor **GET** responses elsewhere may not yet embed `geo_location`. **Tasks** are not filtered by proximity. **Web** has no dedicated nearby/explore page.

## Requirements

### Functional

- [ ] **R1 ŌĆö Geocoding adapter**: Port or re-implement the Living Codex **GeocodingService** contract: try **OpenCage** when `OPENCAGE_API_KEY` is set; on failure or rate limit, fall back to **Nominatim** (respecting [Usage Policy](https://operations.osmfoundation.org/policies/nominatim/): User-Agent, rate limits); return structured `{ latitude, longitude, label, source, confidence }` or a typed error. No silent substitution of wrong city.
- [ ] **R2 ŌĆö Profile enrichment**: Contributor read models used by API consumers (e.g. `GET /api/contributors/{id}` if present, portfolio surfaces, briefs) include **optional** `location_summary` (city/region/country only) when visibility allows; never expose raw precision beyond stored rounding (two decimal places Ōēł ~1 km).
- [ ] **R3 ŌĆö Ideas**: `GET /api/ideas` (or dedicated sub-resource) supports `near_lat`, `near_lon`, `radius_km` **or** `location_query` (string resolved via GeocodingService) to filter/rank ideas by contributor geography (ideas inherit contributor location when author is linked; optional future: per-idea pin).
- [ ] **R4 ŌĆö News**: Extend regional news beyond keyword overlap: optional path `GET /api/news/resonance/proximity?lat=&lon=&radius_km=` that intersects ingested articles with geocoded datelines/places when metadata exists; keep existing `.../local` for text-only fallback.
- [ ] **R5 ŌĆö Tasks**: Agent/task listing endpoints support the same proximity filters (`near_lat`, `near_lon`, `radius_km`) where task records carry assignee/creator contributor id and contributor has public/contributors_only location.
- [ ] **R6 ŌĆö Nearby collaborators UX**: Web exposes at least **`/explore/nearby`** (or `/collaborate/nearby`) with browser geolocation (opt-in) or manual city search, calling `GET /api/nearby` and rendering contributor + idea cards.

### Non-functional

- [ ] **N1 ŌĆö Privacy**: Default visibility remains `contributors_only`; `private` excludes from all proximity and public profile fields.
- [ ] **N2 ŌĆö Rate limits**: Geocoding calls are cached (normalized query ŌåÆ result TTL, e.g. 24h) to protect providers and cost.
- [ ] **N3 ŌĆö Observability**: Expose proof signals (see **Proof and improvement over time** below).

## Data model

### Geocode result (new)

```yaml
GeocodeResult:
  query_normalized: string
  latitude: float
  longitude: float
  label: string
  source: enum [opencage, nominatim, cache]
  confidence: float  # 0.0ŌĆō1.0 when provided by provider
```

### Contributor (extension)

Existing `geo_location` blob on graph node remains source of truth. Add optional **denormalized** `location_summary` on API DTOs for list views (city, country only).

### Cache (new)

- Key: `sha256(normalized_query + provider_version)`
- Value: `GeocodeResult` + `cached_at` (ISO 8601 UTC)

## Verification Scenarios

### Scenario 1 ŌĆö Full location CRUD cycle (contributor)

- **Setup**: API running; contributor `contrib-a` exists (create via existing contributor flow if needed). No prior location.
- **Action**:  
  `curl -sS -X PATCH "$API/api/contributors/contrib-a/location" -H "Content-Type: application/json" -d '{"city":"Boulder","region":"CO","country":"US","latitude":40.02,"longitude":-105.27,"visibility":"public"}'`  
  Then: `curl -sS "$API/api/contributors/contrib-a/location"`
- **Expected**: First returns **200** with rounded lat/lon and metadata; second returns **200** with same city/country and visibility `public`.
- **Edge**: `PATCH` with invalid latitude **90.1** ŌåÆ **422** validation error. `GET` after `DELETE` ŌåÆ **404** with clear detail (not 500).

### Scenario 2 ŌĆö Nearby discovery (create-read for aggregate)

- **Setup**: Two contributors `near-a` and `far-b` with locations at (40.0, -105.0) and (48.0, -122.0); at least one idea authored by `near-a`.
- **Action**: `curl -sS "$API/api/nearby?lat=40.015&lon=-105.28&radius_km=50&limit=20"`
- **Expected**: Response JSON includes `near-a` in `contributors`; `ideas` contains the idea tied to `near-a`; `far-b` absent. Fields include `distance_km` ascending for contributors.
- **Edge**: `radius_km=0` or negative ŌåÆ **422**; `lat=999` ŌåÆ **422**.

### Scenario 3 ŌĆö Geocode resolve (forward geocoding) ŌĆö after implementation

- **Setup**: `OPENCAGE_API_KEY` set in staging, or Nominatim-only fallback.
- **Action**: `curl -sS -X POST "$API/api/geocode" -H "Content-Type: application/json" -d '{"query":"Boulder, Colorado, USA"}'`
- **Expected**: **200** with `latitude`, `longitude`, `label`, `source` in `{ opencage, nominatim, cache }`.
- **Edge**: Empty `query` ŌåÆ **422**; nonsense string with no results ŌåÆ **404** or **200** with `detail` field explaining no match (spec must pick one and test it).

### Scenario 4 ŌĆö Ideas filtered by proximity ŌĆö after implementation

- **Setup**: Ideas and contributors as in Scenario 2; idea linked to nearby contributor.
- **Action**: `curl -sS "$API/api/ideas?near_lat=40.015&near_lon=-105.28&radius_km=100"`
- **Expected**: JSON list contains the nearby idea before distant ones, or only nearby ideas per filter contract.
- **Edge**: Missing `near_lat` but `near_lon` present ŌåÆ **422**.

### Scenario 5 ŌĆö Health and error handling

- **Setup**: Toggle OpenCage key absent; Nominatim allowed.
- **Action**: `curl -sS "$API/api/health/geolocation"`
- **Expected**: JSON shows `opencage_configured: false`, `nominatim_reachable: true` (or explicit false with reason), no secrets.
- **Edge**: Simulate provider timeout in test env ŌĆö geocode endpoint returns **503** with retry hint, not empty 200 with zeros.

## Out of Scope (MVP)

- Precise GPS tracking, turn-by-turn, or background continuous location on mobile.
- Legal jurisdiction compliance review beyond documented provider ToS (follow-up with counsel for EU/GDPR if storing location at scale).
- Neo4j spatial indexes (can be follow-up if O(n) scan becomes hot).

## Risks and Assumptions

- **Risk**: Nominatim abuse or blocking if User-Agent/rate limits ignored ŌĆö **Mitigation**: cache aggressively, single shared User-Agent string with contact, prefer OpenCage in production.
- **Risk**: Ideas/tasks without author linkage cannot be geo-filtered ŌĆö **Mitigation**: document and return unlocated items only when `include_unlocated=true`.
- **Assumption**: Graph contributor nodes continue to store `geo_location` under existing `_GEO_KEY` for backward compatibility.

## Known Gaps and Follow-up Tasks

- Per-idea geographic pins independent of contributor home location.
- Federated instances: cross-instance proximity with privacy-preserving aggregates only.

## Failure/Retry Reflection

- **Failure mode**: OpenCage outage during spike ŌĆö **Blind spot**: cache cold miss storm ŌĆö **Next action**: exponential backoff + short-circuit to Nominatim with stricter rate limit.
