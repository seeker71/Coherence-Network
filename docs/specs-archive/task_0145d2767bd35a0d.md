## Summary

The Coherence Network web UI currently relies on long vertical scrolling for almost every page.
This creates three compounding problems:

1. **Context loss** — users scroll past key metadata without realising it relates to the section above.
2. **Mobile unusability** — long-scroll pages on small screens require excessive thumb travel and hide actions below the fold.
3. **Mode blindness** — the same page is shown to a first-time visitor (who needs a guided, vocabulary-simplified view) and an experienced contributor (who wants IDs, raw JSON, and direct API links).

This spec defines the complete overhaul to fix all three problems in one coherent pass:

- Replace long-scroll sections with **shadcn/ui Tabs** on every detail and list page.
- Add a **mobile-first layout** with a bottom tab bar, swipeable cards, and collapsible sections.
- Add **novice mode** (default) with simplified vocabulary and guided tooltips.
- Add **expert mode** toggled per-contributor, showing technical fields and raw JSON.
- Persist mode and layout preferences via the existing `GET/PUT/PATCH /api/preferences/ui` endpoints.

### What "Done" Looks Like

- Idea detail page (`/ideas/<id>`) renders six tabs: Overview | Specs | Tasks | Contributors | Edges | History.
- Ideas list page (`/ideas`) has three view modes switchable via segmented control: Cards | Table | Graph.
- Primary nav uses the five primary tabs + a "More" dropdown for secondary items.
- On viewports ≤ 640 px, a bottom tab bar replaces the top header nav for the five primary routes.
- On mobile, cards in the Ideas list are swipeable left/right.
- Long sections (e.g. full description, open questions) are collapsible on mobile.
- A `?mode=expert` query param (or toggle stored in preferences) activates expert mode.
- Expert mode surfaces: idea/concept/contributor IDs, raw JSON toggle button, API deep-link badges.
- Novice mode hides: `id` fields, numeric scores except confidence, `resistance_risk`, raw JSON, API links.
- Novice mode shows: friendly labels, guided tooltip overlays on first visit, simplified status labels.
- `GET /api/preferences/ui?contributor_id=<id>` returns the full preference object (already implemented).
- `PUT /api/preferences/ui` fully replaces preferences; `PATCH /api/preferences/ui` partially updates them.
- A shadcn `Tabs` component exists at `web/components/ui/tabs.tsx`.

## Data Model

### UIPreferences (already implemented in `api/app/routers/ui_preferences.py`)

```typescript
interface UIPreferences {
  contributor_id: string;          // "anonymous" for unauthenticated visitors
  expert_mode: boolean;            // false = novice (default)
  nav_layout: "top" | "bottom_bar"; // "top" default; auto-switch on mobile
  ideas_view: "cards" | "table" | "graph"; // ideas list view mode
  idea_detail_tab:
    | "overview" | "specs" | "tasks"
    | "contributors" | "edges" | "history"; // last active tab
  swipeable_cards: boolean;        // touch swipe on mobile
  collapsible_sections: boolean;   // true = auto-collapse long sections on mobile
  show_tooltips: boolean;          // true = show guided tooltips (novice mode only)
}
```

No database migration is required at this time — the API stores preferences in-memory (with a
`TODO: persist to PostgreSQL` comment). A follow-up spec will add DB persistence.

## API Specification

All endpoints already exist. This spec only clarifies expected behaviour and adds new query-param
semantics that the implementation must honour.

### `GET /api/preferences/ui`

```
GET /api/preferences/ui?contributor_id={contributor_id}
```

- Returns `200 UIPreferences` for the given `contributor_id`.
- If no preferences exist, returns the system defaults (all fields present, no 404).
- `contributor_id` defaults to `"anonymous"` if omitted.

### `PUT /api/preferences/ui`

```
PUT /api/preferences/ui
Content-Type: application/json

{
  "contributor_id": "alice",
  "expert_mode": true,
  "nav_layout": "bottom_bar",
  "ideas_view": "table",
  "idea_detail_tab": "specs",
  "swipeable_cards": true,
  "collapsible_sections": true,
  "show_tooltips": false
}
```

- Returns `200 UIPreferences` (full updated object).
- Returns `422` if `contributor_id` is missing or blank.
- Fully replaces all stored preferences for the contributor.

### `PATCH /api/preferences/ui`

```
PATCH /api/preferences/ui?contributor_id=alice
Content-Type: application/json

{ "expert_mode": true }
```

- Returns `200 UIPreferences` with only the supplied fields updated.
- Returns `422` if `contributor_id` query param is missing.
- Other fields remain unchanged.

### `DELETE /api/preferences/ui`

```
DELETE /api/preferences/ui?contributor_id=alice
```

- Returns `204 No Content`.
- Reverts stored preferences; next `GET` returns defaults.

## Component Specifications

### `web/components/ui/tabs.tsx` — shadcn/ui Tabs

Implement using `@radix-ui/react-tabs` (already in `package.json` via shadcn).
Export: `Tabs`, `TabsList`, `TabsTrigger`, `TabsContent`.

```tsx
// Usage on idea detail page:
<Tabs defaultValue="overview">
  <TabsList>
    <TabsTrigger value="overview">Overview</TabsTrigger>
    <TabsTrigger value="specs">Specs</TabsTrigger>
    <TabsTrigger value="tasks">Tasks</TabsTrigger>
    <TabsTrigger value="contributors">Contributors</TabsTrigger>
    <TabsTrigger value="edges">Edges</TabsTrigger>
    <TabsTrigger value="history">History</TabsTrigger>
  </TabsList>
  <TabsContent value="overview">…</TabsContent>
  {/* etc */}
</Tabs>
```

Styling: tabs list sits below the idea title, sticky at `top-14` (below the fixed site header).
Active tab uses `bg-primary/10 text-primary` ring; inactive uses muted.

### `web/components/ideas/IdeaDetailTabs.tsx` — Six-Tab Layout

| Tab | Contents |
|-----|---------|
| **Overview** | Description, free energy score, confidence bar, potential/actual value, manifestation status, stage badge, open questions (collapsible on mobile) |
| **Specs** | Links to associated spec files; DSSS spec builder; raw JSON toggle (expert only) |
| **Tasks** | Task quick-create; task list with status badges |
| **Contributors** | Contributor avatars, stakes, attribution; raw `contributor_id`s (expert only) |
| **Edges** | Related ideas graph mini-view; edge type labels; edge IDs (expert only) |
| **History** | Audit log entries, timestamps, agent attribution |

Expert-mode-only items are wrapped in `{isExpert && <…>}`.

### `web/components/ideas/IdeasViewSwitcher.tsx` — View Mode Switcher

```tsx
<IdeasViewSwitcher
  value={ideasView}           // "cards" | "table" | "graph"
  onChange={setIdeasView}     // patches preferences API + updates local state
/>
```

- **Cards** (default): existing card grid, swipeable on mobile.
- **Table**: compact sortable table with columns: Name | Stage | Confidence | Value Gap | Status.
- **Graph**: embeds the `/graph` page in an `<iframe>` or re-uses the graph component.

### `web/components/BottomTabBar.tsx` — Mobile Bottom Nav

Shown when `window.innerWidth ≤ 640` (or via `useMobileBreakpoint`).
Five tabs: Ideas | Concepts | Contributors | News | Tasks (matching the primary nav).
Each tab has an icon + label. Active tab highlighted in primary colour.
Positioned `fixed bottom-0 left-0 right-0 z-50 bg-background border-t border-border`.

### `web/components/ModeToggle.tsx` — Novice / Expert Toggle

- Renders as a toggle button: "Expert mode" with a toggle switch.
- Reads initial state from `usePreferences()` hook.
- On toggle, calls `PATCH /api/preferences/ui?contributor_id=<id>` with `{ expert_mode: !current }`.
- Placed in the site header (desktop: right side) and in a Settings panel (mobile).
- Visual: when expert mode is on, a small "EXP" badge appears in the header.

### `web/lib/usePreferences.ts` — React Hook

```typescript
function usePreferences(contributorId: string = "anonymous"): {
  prefs: UIPreferences;
  patch: (update: Partial<UIPreferences>) => Promise<void>;
  reset: () => Promise<void>;
  loading: boolean;
}
```

- `GET /api/preferences/ui?contributor_id=<id>` on mount.
- `patch()` calls `PATCH /api/preferences/ui?contributor_id=<id>`.
- `reset()` calls `DELETE /api/preferences/ui?contributor_id=<id>`.
- Local state updated optimistically; reverted on API error.

## Mobile-First Layout Rules

| Breakpoint | Nav | Ideas List | Idea Detail |
|-----------|-----|-----------|------------|
| `< 640px` (mobile) | Bottom tab bar (5 items); top header hidden | Swipeable card deck | Tabs with horizontal scroll on TabsList |
| `640–1024px` (tablet) | Top header primary tabs + More dropdown | Card grid (2 col) | Tabs |
| `> 1024px` (desktop) | Top header primary tabs + More dropdown | Card grid (3+ col) or Table | Tabs |

### Swipeable Cards (mobile)

- Touch-swipe left/right through idea cards (no paging dots required initially).
- Each swipe moves to the next/previous idea.
- Implemented with CSS `scroll-snap` or a minimal touch event handler (no heavy library required).

### Collapsible Sections

- On mobile, sections > 200 characters in the Overview tab are collapsed by default with a "Show more" toggle.
- Implemented by `<CollapsibleSection maxChars={200}>…</CollapsibleSection>`.
- State is local (not persisted); re-collapsing on page navigation is acceptable.

## Expert vs. Novice Mode Field Map

### Fields shown in **expert mode only**

| Location | Field |
|----------|-------|
| Idea detail — Overview | `idea.id` (monospace badge) |
| Idea detail — Overview | `resistance_risk` score |
| Idea detail — Overview | Raw JSON toggle (renders `<pre>` with full idea JSON) |
| Idea detail — any tab | "API: /api/ideas/<id>" link badge |
| Idea detail — Contributors | `contributor_id` values |
| Idea detail — Edges | Edge `id` values |
| Ideas list — Table view | `id` column |
| Concepts page | `concept.id` |

### Fields hidden / renamed in **novice mode**

| Field (raw) | Novice label | Visibility rule |
|-------------|-------------|----------------|
| `resistance_risk` | *(hidden)* | Hidden |
| `free_energy_score` | "Priority Score" | Visible |
| `manifestation_status` | "Status" (humanized string) | Visible |
| `idea.id` | *(hidden)* | Hidden |
| JSON toggle | *(hidden)* | Hidden |
| API link badge | *(hidden)* | Hidden |

### Guided Tooltips (novice mode, `show_tooltips: true`)

First time a user visits a page, tooltip overlays explain:

| Target | Tooltip text |
|--------|-------------|
| "Priority Score" badge | "How urgently this idea needs attention — higher is more urgent." |
| Confidence bar | "How confident the network is that this idea will deliver value." |
| Stage badge | "Where this idea is in its lifecycle: idea → prototype → shipped." |
| Value Gap | "The difference between this idea's potential and what it has delivered so far." |

Tooltips are shown once per session (localStorage flag). They can be permanently dismissed via `PATCH /api/preferences/ui` setting `show_tooltips: false`.

## Verification Scenarios

### Scenario 1 — GET defaults, then enable expert mode

**Setup**: Fresh contributor `test-user-ux-01` with no stored preferences.

**Actions and expected results**:

```bash
API=https://api.coherencycoin.com

# 1. Get defaults
curl -s "$API/api/preferences/ui?contributor_id=test-user-ux-01" | jq .
# Expected: HTTP 200, expert_mode=false, ideas_view="cards", nav_layout="top",
#           show_tooltips=true, idea_detail_tab="overview"

# 2. Enable expert mode
curl -s -X PATCH "$API/api/preferences/ui?contributor_id=test-user-ux-01" \
  -H "Content-Type: application/json" \
  -d '{"expert_mode": true}' | jq .
# Expected: HTTP 200, expert_mode=true, all other fields unchanged

# 3. Verify persistence
curl -s "$API/api/preferences/ui?contributor_id=test-user-ux-01" | jq .expert_mode
# Expected: true

# 4. Reset
curl -s -X DELETE "$API/api/preferences/ui?contributor_id=test-user-ux-01"
# Expected: HTTP 204

# 5. Confirm reset
curl -s "$API/api/preferences/ui?contributor_id=test-user-ux-01" | jq .expert_mode
# Expected: false (defaults restored)
```

**Edge case**: `PATCH /api/preferences/ui` without `contributor_id` param returns `422` with detail `"contributor_id query param is required"`.

## Risks and Assumptions

| # | Risk | Likelihood | Mitigation |
|---|------|-----------|-----------|
| R1 | `@radix-ui/react-tabs` not in `package.json` | Medium | Add via `pnpm add @radix-ui/react-tabs` in web package step |
| R2 | In-memory preference store lost on API restart | High (known) | Acceptable for MVP; follow-up spec will add PostgreSQL persistence |
| R3 | Bottom tab bar conflicts with existing sticky footer on some pages | Low | Audit pages with `position: fixed` footer elements; add `pb-16` to main content on mobile |
| R4 | Swipeable cards conflicting with browser back gesture on mobile | Medium | Use `touch-action: pan-y` on card container; swipe is horizontal only |
| R5 | Graph view mode in Ideas list requires heavy iframe or graph library | Medium | Acceptable to stub with a link to `/graph?filter=ideas` for initial implementation |
| R6 | Tooltip overlay z-index conflicts with sticky tab bar | Low | Set tooltip z-index to `z-[200]`, above sticky elements |
| R7 | `contributor_id` is currently unauthenticated (relies on client-supplied string) | High | Acceptable for MVP; full auth is a separate spec. Anonymous defaults are safe. |
