# MVP Pilot Onboarding Checklist

Use this checklist for every new project intake to ensure quality, alignment, and trust.

## Phase 1: Intake and Qualification

- [ ] **Discovery Call Completed:** Mission alignment and near-term needs understood.
- [ ] **Project Overview Received:** One-pager or deck explaining the project's goal.
- [ ] **Target Outcome Defined:** What does "success" look like in 90 days?
- [ ] **Qualification Score:** Project meets minimum fit criteria (Mission, Need, Capacity).

## Phase 2: Trust and Security Setup

- [ ] **Identity Verified:** Key team members or organization verified.
- [ ] **AI Security Policy Review:** Team understands and agrees to our AI-assisted operations policy.
- [ ] **Communication Channel Set:** Shared Slack/Discord or dedicated email thread established.

## Phase 3: Technical / Operational Mapping

- [ ] **Ideas/Specs Drafted:** High-level project goals broken down into Coherence Network "Ideas".
- [ ] **Initial ROI Estimate:** Potential value and estimated cost (CC) recorded for the pilot.
- [ ] **Evidence Sources Identified:** How will we verify progress? (GitHub, Docs, Public Links).

## Phase 4: Commitment and Launch

- [ ] **Pilot Agreement Signed/Agreed:** Simple terms for the collaboration.
- [ ] **First Milestone Defined:** Clear, measurable target for the first 30 days.
- [ ] **Funder Matching (if applicable):** Project introduced to 1+ relevant funders in the network.
- [ ] **Dashboard Entry Created:** Project added to our active tracking dashboard.

## Phase 5: Ongoing Review

- [ ] **Weekly Check-in Scheduled:** Cadence for progress updates.
- [ ] **Quality Review Gate:** Criteria for "Passing" each milestone defined.
- [ ] **Feedback Loop:** Monthly review with the project team to refine the pilot.
