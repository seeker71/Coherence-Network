## Summary

Package and publish the Coherence Network MCP server as both an **npm package**
(`coherence-mcp-server`) and a **Python PyPI package** (`coherence-mcp`) so it
appears on the Official MCP Registry, Smithery, Glama, PulseMCP, and other
discovery platforms. Add a `server.json` manifest compatible with
`mcp-publisher`. Automate publish via GitHub Actions so new releases propagate
without manual intervention.

The npm package (`coherence-mcp-server@0.3.1`) is already live on npm.
This spec formalises the npm packaging, adds the missing PyPI package, aligns
the `server.json` manifest with the official MCP Registry schema, and creates
automated release pipelines for both registries. It also introduces a
`/api/discovery/publish-status` endpoint so proof of publication is machine-
readable and testable over time.

---

## Background and Motivation

The Coherence Network MCP server exists and works (Node.js, `mcp-server/`; Python,
`api/mcp_server.py`). However:

1. **No PyPI package** — the Python server cannot be installed with `pip install coherence-mcp`.
2. **No automated publish pipeline** — npm releases are manual; there is no
   guarantee the version in `package.json` matches what is on npm.
3. **`server.json` is version-stale** — it still declares `version: "0.2.0"` while
   `package.json` is at `0.3.1`.
4. **No proof layer** — there is no API endpoint that returns actual npm/PyPI
   download counts, so whether discovery is working is invisible.

This spec fixes all four problems.

## Data Model

### `PackageRegistryStatus`

```python
class PackageRegistryStatus(BaseModel):
    registry: Literal["npm", "pypi"]
    package_name: str
    published_version: str | None      # latest version on registry, or None
    local_version: str                  # version in package.json / pyproject.toml
    versions_in_sync: bool              # published_version == local_version
    weekly_downloads: int | None        # from npm/PyPI API, None if unavailable
    total_downloads: int | None         # PyPI lifetime, or None
    last_published_at: datetime | None
    registry_url: str                   # direct link to package page
    source: Literal["live", "cached", "unavailable"]
    fetched_at: datetime | None
    error: str | None = None
```

### `PublishStatus`

```python
class PublishStatus(BaseModel):
    npm: PackageRegistryStatus
    pypi: PackageRegistryStatus
    server_json_version: str            # version field in mcp-server/server.json
    server_json_in_sync: bool           # server.json version == npm published_version
    smithery_listed: bool | None        # True/False/None (None = check failed)
    glama_listed: bool | None
    pulsemcp_listed: bool | None
    all_registries_listed: bool         # True only if all 3 are True
    summary: str                        # human-readable one-liner
    fetched_at: datetime
```

## GitHub Actions Workflows

### `.github/workflows/publish-npm.yml`

```yaml
name: Publish npm
on:
  push:
    tags: ["v*"]

jobs:
  publish-npm:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: "20"
          registry-url: "https://registry.npmjs.org"
      - name: Verify version sync
        run: |
          TAG=${GITHUB_REF#refs/tags/v}
          PKG=$(node -p "require('./mcp-server/package.json').version")
          if [ "$TAG" != "$PKG" ]; then
            echo "ERROR: tag $TAG != package.json version $PKG"
            exit 1
          fi
      - name: Install and publish
        working-directory: mcp-server
        run: |
          npm ci --omit=dev
          npm publish --access public
        env:
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}
```

### `.github/workflows/publish-pypi.yml`

```yaml
name: Publish PyPI
on:
  push:
    tags: ["v*"]

jobs:
  publish-pypi:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
      - name: Verify version sync
        run: |
          TAG=${GITHUB_REF#refs/tags/v}
          PKG=$(python3 -c "import tomllib; t=tomllib.load(open('mcp-server-python/pyproject.toml','rb')); print(t['project']['version'])")
          if [ "$TAG" != "$PKG" ]; then
            echo "ERROR: tag $TAG != pyproject.toml version $PKG"
            exit 1
          fi
      - name: Build and publish
        working-directory: mcp-server-python
        run: |
          pip install hatch
          hatch build
          pip install twine
          twine upload dist/*
        env:
          TWINE_USERNAME: __token__
          TWINE_PASSWORD: ${{ secrets.PYPI_TOKEN }}
```

## How This Proves It Is Working

The proof must be machine-readable and cumulative:

1. **`GET /api/discovery/publish-status`** — single call returns:
   - `npm.versions_in_sync: true` → latest code is published to npm
   - `pypi.versions_in_sync: true` → Python package is published to PyPI
   - `server_json_in_sync: true` → manifest is not stale
   - `smithery_listed: true` / `glama_listed: true` → discoverable on top platforms

2. **Download counts** — `npm.weekly_downloads` and `pypi.total_downloads` are
   returned in every response. A monotonically increasing weekly downloads
   figure is direct proof of real-world adoption.

3. **`npx coherence-mcp-server --version`** returns the published version,
   proving the npm binary works.

4. **`pip install coherence-mcp && python -m coherence_mcp --help`** exits 0,
   proving the Python package installs and runs.

5. **CI badge** — `publish-npm.yml` and `publish-pypi.yml` status badges on
   `README.md` give at-a-glance publish health.

### Scenario 4 — `/api/discovery/publish-status` returns all fields and handles failures gracefully

**Setup:** API is running. PyPI and npm APIs are reachable.

**Action (happy path):**
```bash
curl -s https://api.coherencycoin.com/api/discovery/publish-status | \
  python3 -c "
import sys, json
d = json.load(sys.stdin)
print('npm sync:', d['npm']['versions_in_sync'])
print('pypi sync:', d['pypi']['versions_in_sync'])
print('server.json sync:', d['server_json_in_sync'])
print('summary:', d['summary'])
"
```

**Expected:** All three `sync` fields are `true`. `summary` contains version numbers.
HTTP 200.

**Action (cache behavior):**
```bash
# First call — live fetch
curl -s "https://api.coherencycoin.com/api/discovery/publish-status" | python3 -c "import sys,json; d=json.load(sys.stdin); print(d['npm']['source'])"
# Expected: "live"

# Second call without refresh — cached
curl -s "https://api.coherencycoin.com/api/discovery/publish-status" | python3 -c "import sys,json; d=json.load(sys.stdin); print(d['npm']['source'])"
# Expected: "cached"

# Force refresh
curl -s "https://api.coherencycoin.com/api/discovery/publish-status?refresh=true" | python3 -c "import sys,json; d=json.load(sys.stdin); print(d['npm']['source'])"
# Expected: "live"
```

**Edge — npm registry down:** Returns HTTP 200 with `npm.source: "unavailable"`,
`npm.error: "upstream timeout"`. `pypi` fields are unaffected.

## Purpose

Package the Coherence Network MCP server as both an npm package (`coherence-mcp-server`) and a Python PyPI package (`coherence-mcp`) with automated publish pipelines, ensuring the server is discoverable on Smithery, Glama, PulseMCP, and the official MCP Registry.

## Requirements

- [ ] `pip install coherence-mcp` succeeds and `python -m coherence_mcp --help` exits 0
- [ ] `mcp-server/server.json` has `version` matching `mcp-server/package.json` and includes a `pypi` package entry
- [ ] `GET /api/discovery/publish-status` returns HTTP 200 with `npm`, `pypi`, `server_json_in_sync`, and `summary` fields
- [ ] When npm or PyPI API is unreachable, endpoint returns HTTP 200 with `source: "unavailable"` — no 500
- [ ] `.github/workflows/publish-npm.yml` and `publish-pypi.yml` validate version sync before publishing
- [ ] All new API code covered by pytest tests in `api/tests/test_publish_status.py`
