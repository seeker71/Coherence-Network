# Specification: Homepage Readability Improvements

## Goal
Improve the readability of the Coherence Network homepage by increasing text contrast and opacity. Body text and interactive element placeholders will be updated to have at least 85% opacity, while maintaining the beautiful "warm amber" aesthetic for headings and decorative elements.

## Files to be Modified
- `web/app/page.tsx`: Homepage content and hero section.
- `web/app/globals.css`: Global styles and color variables.
- `web/components/idea_submit_form.tsx`: Idea submission form with low-contrast placeholders.

## Acceptance Criteria
- [ ] Body text (descriptions, stats labels, step descriptions) has an opacity of at least 0.85 relative to the foreground color.
- [ ] Form placeholders have an opacity of at least 0.85 (up from 0.40/0.50).
- [ ] Headings (H1, H2, H3) retain their current styling to preserve the "ambient aesthetic".
- [ ] Stats numbers (the values) remain highly legible (100% opacity foreground).
- [ ] The "soft ambient glow" and background gradients are preserved.

## Technical Details
- Use Tailwind's opacity modifiers (e.g., `text-foreground/85`) to ensure consistent contrast.
- Prefer `text-foreground/85` over `text-muted-foreground` for primary descriptive text if `muted-foreground` is still too dim.
- Update `IdeaSubmitForm` to use `placeholder:text-muted-foreground/85` or `placeholder:text-foreground/70` as needed for legibility.

## Open Questions Addressed
- **Light mode toggle:** The project currently focuses on a "warm organic dark mode". A light mode toggle is not requested at this stage, but the contrast fix will make the dark mode universally accessible.
- **Proof of work:** Legible stats already show "ideas alive" and "value created", but we can make the "resonance" and "recent activity" sections clearer by improving their typography and contrast.

## Verification Scenarios

### Scenario 1: Hero Section Legibility
- **Setup:** Open the homepage (`/`) on a device.
- **Action:** Inspect the hero description ("A pattern you noticed...").
- **Expected result:** The text color should have an opacity of at least 0.85 (e.g., `text-foreground/85` or `text-muted-foreground` if it's bright enough). It should be clearly readable against the dark background.
- **Edge case:** Even with background glows (ambient light), the text remains sharp.

### Scenario 2: Form Placeholder Clarity
- **Setup:** Focus on the "Share your idea" textarea.
- **Action:** Look at the placeholder text ("I think there should be a way to...").
- **Expected result:** The placeholder text should be significantly more visible than before (at least 0.85 opacity modifier if using muted-foreground, or equivalent).
- **Edge case:** Placeholder text must be distinguishable from actual user input (usually by being slightly dimmer, but still readable).

### Scenario 3: Stats and Activity Feed
- **Setup:** View the stats section (ideas alive, value created).
- **Action:** Compare the label ("ideas alive") with the number.
- **Expected result:** Both are clearly visible. The label should be slightly softer than the number but still very readable (0.85 opacity).
- **Edge case:** On smaller screens, the text remains legible despite smaller font sizes.
