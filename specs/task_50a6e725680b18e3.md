## Summary

Spec 180 establishes submission readiness for six registries (Smithery, Glama,
PulseMCP, mcp.so, skills.sh, askill.sh) and introduces ephemeral stat caching.
This spec adds the **proof layer**: persistent time-series install/download count
storage in PostgreSQL, a scheduled weekly harvest job, trend API endpoints, and
a dashboard that makes growth visible and auditable over time.

The open question — *"How can we show whether it is working?"* — is answered by
a monotonically increasing `total_installs` time series. If the series is flat
or absent, there is no proof of discovery. If it rises, discovery is working.

**Six registries remain the target:**

| Registry | Category | Public Stats API |
|----------|----------|-----------------|
| Smithery | mcp | Yes (`/registry/packages/{name}/stats`) |
| Glama (awesome-mcp-servers) | mcp | GitHub repo stars |
| PulseMCP | mcp | Yes (`/servers`) |
| mcp.so | mcp | None (N/A) |
| skills.sh | skill | None (N/A) |
| askill.sh | skill | None (N/A) |

**Core requirement:** `summary.core_requirement_met` is `true` when ≥ 5 registries
report `submission_ready`.

## Architecture

### Layer 1 — Submission readiness (spec 180, already implemented)

```
GET /api/discovery/registry-submissions
  → registry_discovery_service.build_registry_submission_inventory()
  → checks asset files on disk
```

### Layer 2 — Live + cached stat fetch (spec 180 new endpoints)

```
GET /api/discovery/registry-stats[?refresh=true&registry_id=X]
  → registry_stats_service.fetch_registry_stats()
  → calls Smithery API, PulseMCP API, GitHub stars API
  → writes .cache/registry_stats/{registry}.json (24-h TTL)
  → writes to registry_stat_snapshots table (this spec)
```

### Layer 3 — Time-series trends (this spec — new)

```
GET /api/discovery/registry-trends[?days=30&registry_id=X]
  → registry_trends_service.get_trends()
  → reads registry_stat_snapshots table
  → computes delta, growth rate, last-updated
```

### Layer 4 — Unified dashboard (this spec — extended)

```
GET /api/discovery/registry-dashboard
  → merges inventory + latest stats + trend direction
  → always returns HTTP 200 even if stats unavailable
```

### Layer 5 — Harvest job (this spec — new)

```
Weekly cron (Sundays 09:00 UTC):
  registry_stats_harvester.harvest_all()
  → equivalent to GET /api/discovery/registry-stats?refresh=true
  → persists all snapshots to PostgreSQL
  → logs result to api.log
```

## Data Model

### PostgreSQL table: `registry_stat_snapshots`

```sql
CREATE TABLE registry_stat_snapshots (
    id            SERIAL PRIMARY KEY,
    registry_id   VARCHAR(64)  NOT NULL,
    registry_name VARCHAR(128) NOT NULL,
    snapshot_at   TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    install_count INTEGER,
    download_count INTEGER,
    star_count    INTEGER,
    source        VARCHAR(32)  NOT NULL,  -- live | cached | unavailable
    raw_response  JSONB,                  -- full upstream response for audit
    error         TEXT
);

CREATE INDEX idx_registry_stat_snapshots_registry_id ON registry_stat_snapshots(registry_id);
CREATE INDEX idx_registry_stat_snapshots_snapshot_at ON registry_stat_snapshots(snapshot_at DESC);
```

class TrendPoint(BaseModel):
    snapshot_at: datetime

## API Changes

### Unchanged endpoint (spec 180 — no contract change)

```
GET /api/discovery/registry-submissions
```

Returns `RegistrySubmissionInventory` with exactly six items:
`smithery`, `glama`, `pulsemcp`, `mcp-so`, `skills-sh`, `askill-sh`.

### New/extended endpoints (this spec)

#### `GET /api/discovery/registry-stats`

Returns `RegistryStatsList`. Fetches from Smithery API, PulseMCP API, and
GitHub stars API. Writes result to PostgreSQL `registry_stat_snapshots`.
Falls back to 24-h file cache if upstream unreachable.

Query params:
- `?refresh=true` — bypass file cache, force live fetch
- `?registry_id=smithery` — filter to one registry

Response codes: `200` always. Per-item `source` field distinguishes live/cached/unavailable.

#### `GET /api/discovery/registry-trends`

Returns `RegistryTrendsResponse`. Reads from `registry_stat_snapshots` table.
Always HTTP 200 even if table is empty (returns empty `time_series`).

Query params:
- `?days=30` — time window (default 30, max 365)
- `?registry_id=smithery` — filter to one registry

#### `GET /api/discovery/registry-dashboard`

Returns `RegistryDashboard`. Merges submission inventory + latest stats +
7-day trend direction. `proof_statement` is a human-readable string like:

```
"5 of 6 registries submission-ready. Smithery: 12 installs (+5 vs 7 days ago). PulseMCP: live data unavailable."
```

Always returns HTTP 200. All count fields may be `null` with no error.

## Stats Service Design

### `api/app/services/registry_stats_service.py`

```python
STAT_TARGETS = [
    StatTarget(
        registry_id="smithery",
        fetch_fn=_fetch_smithery_stats,  # GET https://registry.smithery.ai/registry/packages/coherence-mcp-server/stats
    ),
    StatTarget(
        registry_id="pulsemcp",
        fetch_fn=_fetch_pulsemcp_stats,  # GET https://pulsemcp.com/api/servers?search=coherence-network
    ),
    StatTarget(
        registry_id="glama",
        fetch_fn=_fetch_github_stars,    # GET https://api.github.com/repos/seeker71/Coherence-Network
    ),
    StatTarget(registry_id="mcp-so",    fetch_fn=None),  # UNAVAILABLE
    StatTarget(registry_id="skills-sh", fetch_fn=None),  # UNAVAILABLE
    StatTarget(registry_id="askill-sh", fetch_fn=None),  # UNAVAILABLE
]
```

#### Cache strategy

- File cache path: `.cache/registry_stats/{registry_id}.json`
- TTL: 24 hours
- Cache miss or `refresh=True`: call `fetch_fn`, write file, persist to PostgreSQL
- Fetch failure: return last file-cached value with `source="cached"`; if no
  file cache exists, return `source="unavailable"` with `error` set
- Persist every non-unavailable result to `registry_stat_snapshots` (deduplication
  by `registry_id` + truncated-to-hour `snapshot_at` to avoid flood inserts)

## Harvest Job Design

### `api/app/jobs/registry_stats_harvester.py`

```python
async def harvest_all(db: AsyncSession) -> dict[str, RegistryStats]:
    """Fetch live stats for all registries and persist snapshots.

    Intended to be called weekly by the scheduler or manually via CLI.
    Returns dict of registry_id -> RegistryStats for observability.
    """
    stats = await registry_stats_service.fetch_registry_stats(refresh=True, db=db)
    logger.info(
        "harvest_all complete",
        extra={"total_installs": stats.summary.total_installs,
               "registries_with_counts": stats.summary.registries_with_counts},
    )
    return {item.registry_id: item for item in stats.items}
```

The harvest job is registered with the existing scheduler (if present) or
invoked via:

```bash
cc run registry-harvest   # via CLI
# or
python3 -m api.app.jobs.registry_stats_harvester
```

## Risks and Assumptions

| Risk | Mitigation |
|------|-----------|
| Smithery stats endpoint URL may differ from assumed path | Verify against Smithery docs before implementation; fall back to `unavailable` if endpoint not found |
| PulseMCP public search API may not include install counts | Use whatever count field is available (`usage_count`, `install_count`); document which field is used |
| GitHub API rate limit (60 req/h unauthenticated) | Weekly harvest is 1 request; well within limit. If token is available, use it |
| Registry PRs (Glama, PulseMCP catalog) may be rejected or delayed | Asset readiness and snapshot storage are independent of PR acceptance; `notes` field tracks PR URL |
| PostgreSQL unavailable during harvest | Harvest logs error, writes file cache only; retried on next harvest cycle |
| `registry_stat_snapshots` grows unboundedly | Add a 90-day retention policy in a follow-up migration |
| skills.sh and askill.sh manifest format requirements unknown | Validate before PR submission; update `SKILL.md` validators if needed |

- Smithery stats endpoint URL may differ — fall back to `unavailable` if not found
- PostgreSQL unavailable during harvest — logs error, writes file cache only
