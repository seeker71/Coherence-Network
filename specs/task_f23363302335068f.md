## Summary

The `/invest` page currently reads like a financial spreadsheet — "Value gap", "Est. cost", "ROI" with a bare
progress bar. This framing optimizes for rational calculation but drains the feeling of participation.
Ideas are living things grown through sustained attention; the UI should reflect that.

This spec replaces the spreadsheet metaphor with a **garden metaphor**: ideas become seeds-to-trees,
growth potential is shown as a visual sprout-to-canopy progression, and the language shifts from
financial jargon to organic nurturing language. The underlying numbers are preserved but rendered as
contextual detail, not the primary visual.

### What Done Looks Like

- `web/app/invest/page.tsx` renders each idea card with a **GrowthStage** visual (emoji + label) instead
  of "Value gap / Est. cost / ROI" columns.
- A new `web/app/invest/GrowthStageBar.tsx` component renders a sprout-to-tree 5-stage progression
  with the current stage highlighted. Stage is derived from `free_energy_score` and
  `manifestation_status`.
- Financial numbers (value gap, estimated cost, ROI multiplier) are visible on hover or in a
  collapsible "Details" section, not in the primary card layout.
- The "Stake" CTA becomes "Nurture" (label change only; href and behavior unchanged).
- The balance section header "Your CC Balance" becomes "Your Garden" with a sub-label explaining
  CC as fertilizer.
- A live counter "X ideas growing" appears in the page header, driven by the existing
  `/api/ideas?limit=60` fetch result count.
- All four tests in `web/tests/invest-garden.spec.ts` pass (Playwright).
- No API changes are required — purely frontend.

## Problem Statement

Current language on `/invest`:
- "Value gap" — finance jargon; implies a deficiency rather than potential.
- "Est. cost" — makes ideas feel like purchase orders.
- "ROI" with a 1.5px progress bar — the bar is too small and too abstract to convey meaning.

This creates a cognitive mismatch: the platform is about nurturing intelligence, but the UI is about
spreadsheet arbitrage. The result is that visitors who might intuitively want to contribute feel
alienated by language that assumes financial literacy over ecological intuition.

## Data Model (Frontend Only)

### `growthStage(idea: IdeaWithScore): 0 | 1 | 2 | 3 | 4`

```typescript
function growthStage(idea: IdeaWithScore): number {
  const score = idea.free_energy_score ?? 0;
  const status = (idea.manifestation_status ?? "").trim().toLowerCase();

  if (status === "validated") return 4;           // Ancient
  if (status === "partial") return 3;             // Tree
  if (score >= 0.75) return 2;                    // Shoot
  if (score >= 0.4)  return 1;                    // Sprout
  return 0;                                        // Seed
}
```

### Stage metadata

```typescript
const STAGES = [

  { icon: "🌲", label: "Ancient", description: "Fully realized and validated" },
];
```

## How We Know It's Working (Proving the Metaphor Lands)

The question "is this working?" requires more than passing tests. Garden metaphors work when users feel
they are tending to something, not picking stocks. We measure success through:

1. **Qualitative signal (short-term)**: Session notes from any contributor who mentions "planted",
   "nurturing", "growing" unprompted in feedback or chat — early sign the language has transferred.

2. **Engagement signal (medium-term)**: Compare `/invest` → `/ideas/:id` click-through rate before
   and after. If the garden framing is more inviting, we expect an increase. Track via existing
   Coherence telemetry or simple server-side log analysis.

3. **Retention signal (medium-term)**: Do contributors who see the garden UI return to check growth?
   Track returning visits to `/invest` per contributor ID in the existing ledger.

4. **Spec self-validation**: The `GrowthStageBar` is itself a claim — that growth is measurable and
   visible. If `free_energy_score` or `manifestation_status` do not meaningfully progress over time
   for staked ideas, the visual metaphor becomes false. A follow-up monitoring spec should alert when
   staked ideas show no stage change after N days.

## Risks and Assumptions

- **Assumption**: `free_energy_score` is a float 0.0–1.0 in the API response. If it is `null` or
  outside range, the `growthStage()` function defaults to Stage 0 safely.
- **Risk**: "Nurture" as a CTA may confuse users who are used to "Stake" from other crypto contexts.
  Mitigation: keep the tooltip or sub-label text mentioning CC tokens; the link behavior is unchanged.
- **Risk**: `<details>` disclosure for numbers may be missed by power users. Mitigation: the summary
  label "Numbers ▾" is explicit; if user research shows confusion, a follow-up can promote it.
- **Assumption**: No A/B testing infrastructure exists; this is a direct replacement. If metrics
  worsen, reverting is a single PR.

## Failure/Retry Reflection

- **Failure mode**: GrowthStageBar stage index off-by-one (e.g., `partial` resolves to stage 4 instead of 3).
  - Blind spot: Stage mapping is defined in one place but rendered in another; easy to drift.
  - Next action: Add a unit test for `growthStage()` that covers all 5 branches explicitly.

- **Failure mode**: `<details>` element breaks existing card layout on mobile.
  - Blind spot: `<details>` expands inline and can push adjacent elements out of flow.
  - Next action: Wrap details in a `w-full` container and test on 375px viewport.

- **Failure mode**: "Your Garden" balance section confuses new users who don't know CC is a token.
  - Blind spot: Garden metaphor is more opaque than "CC Balance" for first-time visitors.
  - Next action: Add a one-line tooltip on "CC" explaining it is Coherence Coins.
