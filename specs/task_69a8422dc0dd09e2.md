## Summary

The `/invest` page currently reads like a financial spreadsheet: *Value gap*, *Est. cost*, *ROI*, and a bare horizontal progress bar. This language positions ideas as risk instruments rather than living possibilities. The metaphor is wrong for the product's soul.

This spec reframes the entire Invest page through a **garden metaphor**: ideas are seeds that need water (CC) and light (attention) to grow into trees (proven, widely-used features). The core numbers remain accessible but are labelled and visualised through the lens of growth rather than arbitrage.

---

## Problem Statement

### Current experience
- "Value gap" — opaque financial jargon; users ask "gap between what?"
- "Est. cost" — implies a fixed contract price; ideas are organic, not quoted
- "ROI" with a thin bar — zero emotional resonance; communicates speculation, not nurturing
- No sense of time or momentum: a 2-year-old dormant idea looks identical to one seeded yesterday

### What's missing
1. **Emotional resonance**: visitors feel like auditors, not gardeners
2. **Growth trajectory**: no visual sense of how far an idea has come or how far it could go
3. **Proof of working**: nothing shows whether an investment is "growing" — no milestone indicators
4. **Stage differentiation**: a seed looks the same as a sapling in the current layout

---

## Goal

Replace spreadsheet vocabulary and the bare ROI bar with a coherent garden metaphor across the entire `/invest` page. Numbers stay accurate and accessible (shown on hover or in secondary text), but the primary reading experience is: *this is a living thing you can water*.

### Success criteria
- A visitor reads the page without seeing "ROI", "Value gap", or "Est. cost" as primary labels
- Each idea card shows a sprout-to-tree stage icon that maps to `manifestation_status`
- The growth bar is replaced by a branching visual that fills left-to-right by growth stage
- Numbers are still reachable: in a `title` attribute, collapsed detail row, or small secondary text
- The page passes accessibility contrast requirements (WCAG AA)

---

## Vocabulary Mapping

| Current label | Garden label | Rationale |
|---|---|---|
| Value gap | Growth potential | How much value could bloom if this idea is fully realised |
| Est. cost | Water needed | CC required to reach full growth — water the plant, not buy an asset |
| ROI (bare bar) | Growth stage (visual progression) | Sprout → sapling → tree: communicates trajectory, not return multiple |
| Stake | Nurture | The act of investing; you nurture an idea, not stake a position |
| Manifestation status | Garden stage | The readable label accompanying the icon |

## Growth Stage System

Map `manifestation_status` (existing API field) to a five-stage growth icon set rendered as inline SVG or Unicode plant emojis with a CSS-animated progression bar:

| Stage index | `manifestation_status` value(s) | Icon | Label |
|---|---|---|---|
| 1 | `"none"`, `""`, `null` | 🌱 | Seed |
| 2 | `"specced"` | 🪴 | Sprout |
| 3 | `"partial"` | 🌿 | Sapling |
| 4 | `"validated"` | 🌳 | Tree |
| 5 | `"deployed"` | 🌲 | Grove |

The existing `humanizeManifestationStatus()` function in `web/lib/humanize.ts` **must be extended** (not replaced) with a `gardenStage(status: string): { icon: string; label: string; index: number }` export. The original function remains for backward compatibility with other pages.

### Growth bar design
Replace the thin 1.5px ROI bar with a **segmented branch bar**:
- 5 equal segments, each corresponding to one growth stage
- Filled segments use a green gradient (`from-emerald-400 to-emerald-600`)
- Unfilled segments use muted/leaf-outline styling
- Segment fill is based on `gardenStage(idea.manifestation_status).index`, not ROI calculation
- A tooltip (via `title` or `aria-label`) on the bar segment shows the underlying ROI number: `"Growth potential: 4.2×"`

## Proof-of-Working Indicator

**Open question answered**: *How can we improve this idea, show whether it is working yet, and make that proof clearer over time?*

Each idea card gains a **"Last watered"** indicator — a relative timestamp showing when the idea last received a contribution event (task completion, CC stake, spec commit, or deployment). This answers "is it growing?" without requiring a chart:

- **Active** (event in last 7 days): green dot + "Active 2 days ago"
- **Growing** (event in last 30 days): yellow dot + "Growing 12 days ago"
- **Dormant** (no event in 30+ days): grey dot + "Dormant — needs attention"
- **Unstarted** (no events ever): no dot, faint "Waiting for first water"

Data source: the API already returns `updated_at` on idea objects. The frontend maps the recency bucket to these states. A future enhancement (tracked below) can use a richer event log, but this spec deliberately uses the available field.

This indicator directly answers the open question by making growth/stagnation visible at a glance, without requiring the user to click through.

## Page Structure Changes

### Header section
- Keep: page title "Invest", subheading copy
- Change: subheading rewritten from current neutral copy to garden framing:
  > *"Every idea is a seed. CC is water. When you nurture an idea, real compute runs against it — and something grows."*

### CC Balance section (`InvestBalanceSection`)
- Rename section heading from "Your CC Balance" to "Your watering can"
- Format: `{total} CC — enough to water {N} seeds` where N = `floor(total / 10)` (10 CC = 1 watering)
- No change to underlying data fetch or storage logic

### Idea card layout
Replace the current 3-column metric grid with:

```
[Stage icon]  Idea name                          [Nurture →]
              Garden stage label • Last watered
              ━━━━━━━━━━━━━━━━ (branch bar)
              Growth potential: 2,400 CC  ·  Needs: 580 CC
```

- **Stage icon** (large, left): 2rem emoji or SVG, communicates immediately
- **Garden stage label**: replaces the current `humanizeManifestationStatus` badge
- **Last watered**: proof-of-working dot + relative time
- **Branch bar**: segmented growth stages, replaces ROI bar
- **Growth potential / Needs**: secondary text, smaller font, muted colour. These are the numbers — accessible but not primary. Replaces "Value gap" and "Est. cost" labels.

### Sort order
Keep current default sort by `computeRoi()` (highest potential first) but rename the invisible concept to `growthPotentialScore()` in code. Visible sort label: "Most ready to grow" (shown as a small label above the list). No new UI control needed.

## Verification Scenarios

### Scenario 1 — Garden vocabulary replaces spreadsheet terms (primary labels)

**Setup**: The Invest page at `/invest` loads with at least one idea in the database.
**Action**: Open `/invest` in a browser. Inspect the rendered HTML.
**Expected result**:
- The strings "Value gap", "Est. cost", "ROI" do NOT appear as visible label text in the DOM
- The strings "Growth potential", "Needs", "Nurture" DO appear
- The stage icon (🌱, 🪴, 🌿, 🌳, or 🌲) appears on each idea card
**Edge case**: If an idea has `manifestation_status = null` or `""`, it renders the 🌱 "Seed" stage without crashing (no undefined error)

### Scenario 2 — Branch bar shows correct stage fill

**Setup**: Database contains three ideas with `manifestation_status` of `"none"`, `"partial"`, and `"validated"` respectively.
**Action**: Load `/invest` and inspect the branch bar on each card.
**Expected result**:
- `"none"` idea: 1 of 5 segments filled (stage index 1)
- `"partial"` idea: 3 of 5 segments filled (stage index 3)
- `"validated"` idea: 4 of 5 segments filled (stage index 4)
- Each filled segment has a green class (e.g. `bg-emerald-*`); unfilled segments are muted
**Edge case**: Unknown `manifestation_status` (e.g. `"archived"`) defaults to stage 1 (seed), renders gracefully

### Scenario 3 — "Last watered" proof-of-working indicator

**Setup**: One idea has `updated_at` within the last 7 days; another has `updated_at` 45 days ago; a third has never been updated (null `updated_at`).
**Action**: Load `/invest`.
**Expected result**:
- Recent idea: green dot, label contains "Active" and a relative time (e.g. "Active 3 days ago")
- Old idea: grey dot, label contains "Dormant"
- Never-updated idea: no dot, faint "Waiting for first water" text
**Edge case**: `updated_at` is an invalid date string → treated as null → renders "Waiting for first water"

### Scenario 4 — Numbers remain accessible in secondary position

**Setup**: Any idea on the Invest page.
**Action**: Inspect the idea card HTML.
**Expected result**:
- The `value_gap` number appears in the card as secondary text (smaller font, muted colour) labelled "Growth potential: X CC"
- The `estimated_cost` number appears labelled "Needs: X CC"
- The underlying ROI multiplier appears in the `title` attribute of the branch bar: e.g. `title="Growth potential: 4.2×"`
- These numbers are not the first or largest text the eye lands on
**Edge case**: `value_gap = 0` and `estimated_cost = 0` → branch bar renders at stage 1 (seed), numbers show "0 CC" without divide-by-zero errors

### Scenario 5 — Watering can balance section

**Setup**: A contributor ID with a known balance (e.g. `alice` with 120 CC) is stored in localStorage.
**Action**: Load `/invest` — the balance section renders.
**Expected result**:
- Section heading reads "Your watering can" (not "Your CC Balance")
- Balance line reads "120 CC — enough to water 12 seeds"
- `floor(120 / 10) = 12` is computed correctly
**Edge case**: Balance of 5 CC → "enough to water 0 seeds" (not a negative or NaN)
**Edge case**: Unknown contributor ID → section shows "Enter your contributor name" prompt; "watering can" label is still shown

## Accessibility Requirements

- All stage icons (emoji) have `aria-label` with the stage name: `aria-label="Seed"` etc.
- Branch bar segments have `role="progressbar"`, `aria-valuenow={stageIndex}`, `aria-valuemin={1}`, `aria-valuemax={5}`
- "Last watered" colour dots have an additional text label (not colour alone) for colour-blind users
- Contrast ratio of secondary number text against card background: ≥ 4.5:1 (WCAG AA)

## Risks and Assumptions

| Risk | Likelihood | Mitigation |
|---|---|---|
| `updated_at` not reliably updated on idea changes | Medium | Current API does update this field on any PUT; document dependency so future API authors know |
| Emoji rendering varies across OS/browser | Low | Use system emoji with known good fallbacks; test on macOS + Windows + mobile |
| Garden vocabulary feels whimsical/unserious to finance-minded users | Low–Medium | Numbers remain fully accessible; the metaphor is evocative, not childish; copy uses calm tone |
| Stage mapping is oversimplified (only 5 states) | Low | Stage 5 ("deployed") not currently returned by API; renders same as stage 4 for now |

## Known Gaps and Follow-up Tasks

1. **Rich event log for "Last watered"**: Currently uses `updated_at` as a proxy. A future enhancement should use a dedicated `GET /api/ideas/{id}/events` endpoint to show the full growth timeline (commits, deploys, stakes) as a sparkline on the card.
2. **Animated growth**: A subtle CSS grow animation when a card first enters the viewport (Intersection Observer) would deepen the metaphor. Out of scope here — keep it simple first.
3. **Garden view toggle**: A future "Garden view" that renders ideas as a literal grid of plants at varying heights, with investment triggering a grow animation. This spec only covers the list view reframe.
4. **Personalised garden**: Show "seeds you watered" vs "seeds others watered" in separate sections. Requires auth/contributor context on the page. Out of scope.
5. **`gardenStage()` for status `"deployed"`**: API does not currently return this value; add the mapping when the API starts returning it.
