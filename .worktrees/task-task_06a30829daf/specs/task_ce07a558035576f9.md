# Spec: Belief Systems Translation Idea Lens Engine (Minimal)

**Spec ID / task:** `task_ce07a558035576f9`  
**Status:** Draft  
**Date:** 2026-03-29

## Summary

Belief Systems Translation Idea Lens Engine is a minimal service that restates an existing
idea through a small set of predefined belief-system lenses so a contributor can understand
the same idea in a frame that matches how they reason. This retry intentionally narrows the
earlier broad concept to the single core outcome: given an idea and a supported lens, return
one translated explanation that preserves the source idea's intent and facts while changing
the framing language.

## Purpose

Coherence Network already stores ideas, but the same idea can sound compelling, vague, or
hostile depending on the reader's worldview. The minimal product need is not a full lens
platform; it is a stable contract for translating one idea through one selected lens on
demand so the network can prove perspective-bridging is possible before adding caching,
analytics, contributor personalization, or UI workflows.

## Requirements

- [ ] `POST /api/ideas/{idea_id}/lens-translation` accepts a supported `lens_id` and returns
      a translated response for an existing idea with HTTP 200.
- [ ] The MVP supports a fixed builtin lens set only: `scientific`, `spiritual`, and
      `systemic`. Lens CRUD is explicitly out of scope.
- [ ] The response includes `idea_id`, `lens_id`, `original_name`, `translated_summary`, and
      `preserved_facts` so clients can show both the framing shift and the unchanged substance.
- [ ] The translation must preserve the source idea's core claim and must not invent new
      commitments, metrics, or implementation promises not present in the original idea text.
- [ ] Unknown `idea_id` returns HTTP 404 and unsupported `lens_id` returns HTTP 422 with a
      clear validation error.
- [ ] The translation endpoint is read-only from the product point of view: it does not
      mutate the stored idea and does not require persistence in MVP.

## API changes

### `POST /api/ideas/{idea_id}/lens-translation`

Request body:

```json
{
  "lens_id": "scientific"
}
```

Response `200 OK`:

```json
{
  "idea_id": "idea_123",
  "lens_id": "scientific",
  "original_name": "Community tool cooperative",
  "translated_summary": "A structured coordination mechanism that improves shared tool reliability through distributed ownership.",
  "preserved_facts": [
    "community members share ownership",
    "the idea concerns tools",
    "the proposal is cooperative rather than centralized"
  ],
  "spec_ref": "task_ce07a558035576f9"
}
```

Response `404 Not Found`:

```json
{ "detail": "Idea not found" }
```

Response `422 Unprocessable Entity`:

```json
{ "detail": "Unsupported lens_id. Allowed values: scientific, spiritual, systemic" }
```

No other API surface is required for MVP. Listing lenses, saving translations, contributor
matching, and ROI endpoints are deferred.

## Data model

```yaml
LensTranslationRequest:
  properties:
    lens_id:
      type: string
      enum: [scientific, spiritual, systemic]

LensTranslationResponse:
  properties:
    idea_id: { type: string }
    lens_id: { type: string }
    original_name: { type: string }
    translated_summary: { type: string }
    preserved_facts:
      type: list[string]
      min_items: 1
    spec_ref: { type: string, const: task_ce07a558035576f9 }
```

Implementation may compute this payload on demand from the stored idea record. No new table
or cache key is required for the minimal release.

## Out of Scope

- Lens creation, update, delete, or operator management APIs.
- Translation caching, persistence, or background regeneration jobs.
- Contributor-specific resonance scoring or belief-profile matching.
- Web UI selectors, dashboards, or analytics endpoints.

## Risks

- Translation quality may drift from the original idea if the implementation treats framing
  as rewriting instead of constrained restatement.
- A lens set that is too broad or too political could create needless ambiguity; the MVP
  should stay with a small, documented builtin set.
- If the source idea lacks enough detail, `preserved_facts` may become thin; the API should
  still return a translation but avoid fabricated specificity.
